# Todo App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Next.js 14 풀스택 할일 앱 — 이메일/비밀번호 인증 + 할일 CRUD (생성/조회/수정/삭제/완료 토글)

**Architecture:** Next.js App Router 단일 레포. API Routes로 REST API 구현, NextAuth v5 Credentials로 세션 관리, Prisma ORM으로 Supabase PostgreSQL 접근. 클라이언트 상태는 TanStack Query(서버 데이터) + Zustand(UI 상태)로 분리.

**Tech Stack:** Next.js 14, TypeScript, NextAuth v5, Prisma, PostgreSQL (Supabase), TanStack Query v5, Zustand, Tailwind CSS, shadcn/ui, pnpm, Vercel

## 기능 정의

### 인증
| 기능 | 설명 |
|------|------|
| 회원가입 | 이메일 + 비밀번호 입력. 중복 이메일 불가, 비밀번호 6자 이상 |
| 로그인 | 이메일 + 비밀번호. 실패 시 에러 메시지 표시 |
| 로그아웃 | 헤더 버튼 클릭 → 세션 삭제 → /login 이동 |
| 인증 보호 | 비로그인 상태에서 /todos 접근 시 /login으로 자동 이동 |
| 로그인 후 리다이렉트 | 로그인 상태에서 /login, /register 접근 시 /todos로 자동 이동 |

### 할일 CRUD
| 기능 | 설명 |
|------|------|
| 전체 피드 조회 | 모든 유저의 할일을 최신순 카드 형태로 표시. 작성자 이메일 표시 |
| 추가 | 상단 입력창에 내용 입력 후 추가 버튼 또는 Enter |
| 완료 토글 | 체크박스 클릭 → done 상태 반전, 완료 시 취소선. **내 글만 가능** |
| 수정 | 연필 아이콘 클릭 → 인라인 편집 모드, 확인/취소 버튼. **내 글만 가능** |
| 삭제 | 휴지통 아이콘 클릭 → 즉시 삭제. **내 글만 가능** |
| 권한 표시 | 수정/삭제 버튼은 내 글에만 렌더링 (다른 사람 글엔 버튼 없음) |

### 댓글 CRUD
| 기능 | 설명 |
|------|------|
| 댓글 조회 | 카드 하단 "댓글 N개" 클릭 시 댓글 목록 펼침/접힘 토글 |
| 댓글 작성 | 댓글 목록 하단 입력창 + 등록 버튼. 로그인 유저 누구나 가능 |
| 댓글 수정 | 연필 아이콘 클릭 → 인라인 편집. **내 댓글만 가능** |
| 댓글 삭제 | 휴지통 아이콘 클릭 → 즉시 삭제. **내 댓글만 가능** |
| 작성자 표시 | 댓글마다 작성자 이메일 표시 |

### 상태 처리
| 상태 | 처리 |
|------|------|
| 로딩 중 | "불러오는 중..." 텍스트 표시 |
| 빈 목록 | "아직 글이 없습니다. 첫 번째로 작성해보세요!" 표시 |
| 빈 댓글 | "첫 번째 댓글을 남겨보세요!" 표시 |
| 에러 | "불러오기 실패" 메시지 표시 |
| 추가/수정 중 | 버튼 disabled 처리 |

```prisma
// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        Int       @id @default(autoincrement())
  email     String    @unique
  password  String
  todos     Todo[]
  comments  Comment[]
  createdAt DateTime  @default(now()) @map("created_at")

  @@map("users")
}

model Todo {
  id        Int       @id @default(autoincrement())
  content   String
  done      Boolean   @default(false)
  user      User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  userId    Int       @map("user_id")
  comments  Comment[]
  createdAt DateTime  @default(now()) @map("created_at")

  @@map("todos")
}

model Comment {
  id        Int      @id @default(autoincrement())
  content   String
  todo      Todo     @relation(fields: [todoId], references: [id], onDelete: Cascade)
  todoId    Int      @map("todo_id")
  author    User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  userId    Int      @map("user_id")
  createdAt DateTime @default(now()) @map("created_at")

  @@map("comments")
}
```

```ts
import NextAuth from 'next-auth'
import Credentials from 'next-auth/providers/credentials'
import bcrypt from 'bcryptjs'
import { prisma } from './prisma'

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [
    Credentials({
      credentials: {
        email: { label: 'Email', type: 'email' },
        password: { label: 'Password', type: 'password' },
      },
      async authorize(credentials) {
        if (!credentials?.email || !credentials?.password) return null

        const user = await prisma.user.findUnique({
          where: { email: credentials.email as string },
        })

        if (!user) return null

        const isValid = await bcrypt.compare(
          credentials.password as string,
          user.password
        )
        if (!isValid) return null

        return { id: String(user.id), email: user.email }
      },
    }),
  ],
  pages: { signIn: '/login' },
  callbacks: {
    jwt({ token, user }) {
      if (user) token.id = user.id
      return token
    },
    session({ session, token }) {
      if (token.id) session.user.id = token.id as string
      return session
    },
  },
})
```

```ts
import { NextResponse } from 'next/server'
import bcrypt from 'bcryptjs'
import { prisma } from '@/lib/prisma'

export async function POST(req: Request) {
  const { email, password } = await req.json()

  if (!email?.trim() || !password) {
    return NextResponse.json(
      { error: '이메일과 비밀번호를 입력해주세요.' },
      { status: 400 }
    )
  }

  if (password.length < 6) {
    return NextResponse.json(
      { error: '비밀번호는 6자 이상이어야 합니다.' },
      { status: 400 }
    )
  }

  const existing = await prisma.user.findUnique({ where: { email } })
  if (existing) {
    return NextResponse.json(
      { error: '이미 사용 중인 이메일입니다.' },
      { status: 409 }
    )
  }

  const hashed = await bcrypt.hash(password, 12)
  const user = await prisma.user.create({
    data: { email, password: hashed },
    select: { id: true, email: true },
  })

  return NextResponse.json(user, { status: 201 })
}
```

```tsx
            <span className={`text-sm ${todo.done ? 'line-through text-muted-foreground' : ''}`}>
              {todo.content}
            </span>
          )}
        </div>
        {isOwner && !isEditing && (
          <div className="flex gap-1 flex-shrink-0">
            <Button size="icon" variant="ghost" onClick={() => setEditingTodoId(todo.id)}>
              <Pencil className="w-4 h-4" />
            </Button>
            <Button size="icon" variant="ghost" onClick={() => deleteTodo(todo.id)} disabled={isDeleting}>
              <Trash2 className="w-4 h-4" />
            </Button>
          </div>
        )}
      </div>
      <CommentSection todoId={todo.id} />
    </li>
  )
}
```

### Task 12: GitHub Actions CI 설정

**Files:**
- Create: `.github/workflows/ci.yml`

PR을 올리거나 `main`에 push할 때 자동으로 테스트를 실행한다. Vercel은 이 워크플로우와 별개로 자체 배포를 진행하지만, 테스트가 어떻게 돌아가는지 GitHub Actions 로그로 직접 확인할 수 있다.

- [ ] **Step 1: 워크플로우 디렉토리 생성**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: .github/workflows/ci.yml 작성**

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    name: Test
    runs-on: ubuntu-latest

    steps:
      - name: 코드 체크아웃
        uses: actions/checkout@v4

      - name: pnpm 설치
        uses: pnpm/action-setup@v3
        with:
          version: 9

      - name: Node.js 설정
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: 의존성 설치
        run: pnpm install --frozen-lockfile

      - name: 타입 체크
        run: pnpm tsc --noEmit

      - name: 테스트 실행
        run: pnpm test --run

      - name: 빌드 확인
        run: pnpm build
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
          NEXTAUTH_SECRET: ${{ secrets.NEXTAUTH_SECRET }}
          NEXTAUTH_URL: http://localhost:3000
```
